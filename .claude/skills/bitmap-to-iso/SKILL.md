# Bitmap → Isometric SVG 변환 스킬

비트맵 레퍼런스 이미지를 구조화된 아이소메트릭 SVG 컴포넌트로 정확하게 옮기는 강제 파이프라인.

**핵심 원칙: 눈대중 금지. 모든 값은 측정에서 나온다.**

---

## 사용 조건

- 입력: 비트맵 레퍼런스 이미지 (PNG/JPG)
- 출력: React SVG 컴포넌트 (isometricGrid.js 유틸 활용)
- 전제: Storybook dev server 실행 중, Playwright MCP 사용 가능

---

## 강제 파이프라인 (단계 건너뛰기 금지)

### Stage 0: 레퍼런스 로드 & 기본 정보

```
필수 출력:
- 이미지 경로
- 이미지 크기 (px)
- 아이소메트릭 요소 개수 (슬래브/프리즘/카드)
- 각 요소 식별 (id, 설명)
```

이미지를 Read 도구로 열고, 무엇이 있는지 **목록화**한다.
"대충 5개 레이어가 있다" (X) → "5개 슬래브: background, spatial, motion, color, typography" (O)

---

### Stage 1: 픽셀 측정 (Playwright Canvas API)

**반드시 Playwright로 프로그래밍적 측정을 수행한다. 시각적 추정 금지.**

```js
// 측정 코드 템플릿 — 반드시 실행
const img = new Image();
img.src = 'file:///path/to/reference.png';
// canvas에 그린 후 getImageData()로 픽셀 분석
```

#### 측정 항목 (전부 필수, 하나라도 빠지면 다음 단계 진입 금지)

| # | 측정 항목 | 방법 | 단위 |
|---|---------|------|------|
| 1 | 전체 캔버스 크기 | 이미지 naturalWidth/Height | px |
| 2 | 각 슬래브의 바운딩 박스 | 밝은 픽셀 영역 스캔 | {x, y, w, h} px |
| 3 | 슬래브 다이아몬드 폭/높이 | 좌→우 최대 수평 거리, 상→하 최대 수직 거리 | px |
| 4 | 슬래브 간 수직 간격 | 인접 슬래브 상단 y 차이 | px |
| 5 | 콘텐츠 요소 바운딩 박스 | 슬래브 내부의 밝은 픽셀 영역 | {x, y, w, h} px |
| 6 | 콘텐츠/슬래브 면적 비율 | 콘텐츠 bbox / 슬래브 bbox | % |
| 7 | 스트로크 두께 | 밝은 픽셀 연속 카운트 (수평/수직) | px |
| 8 | 전체 구성 내 여백 | 슬래브 스택의 viewBox 내 위치 | px (상/하/좌/우) |

#### 측정 결과 기록 형식 (반드시 이 형식으로 출력)

```
## 측정 결과

### 캔버스
- 원본 크기: {W} × {H} px
- SVG viewBox 환산: {vbW} × {vbH} (비율: {ratio})

### 슬래브 #{n}: {id}
- 스크린 bbox: x={}, y={}, w={}, h={}
- 다이아몬드: 폭={}, 높이={}
- 바디 높이: {}px
- 콘텐츠 bbox: x={}, y={}, w={}, h={}
- 콘텐츠/슬래브 비율: {}%
- 스트로크 두께: {}px

### 레이어 간격
- #{0}→#{1}: {}px
- #{1}→#{2}: {}px
- ...
```

**이 단계에서 Playwright 코드를 실행하지 않으면 Stage 2로 진입할 수 없다.**

---

### Stage 2: 역투영 — 스크린 좌표 → 아이소메트릭 flat 좌표

스크린 픽셀 측정값을 아이소메트릭 flat 좌표계로 변환한다.

#### 변환 공식

아이소메트릭 변환 행렬: `matrix(1, 0.5, -1, 0.5, cx, topY)`

스크린 (sx, sy) → flat (fx, fy):
```
fx = (sx - cx + (sy - topY))
fy = ((sy - topY) - (sx - cx))
// 주의: 이 역변환은 top face 위의 점에만 유효
```

다이아몬드 치수에서 flat 치수 도출:
```
diamond_width = fw + fd  (스크린 수평 폭)
diamond_height = (fw + fd) / 2  (스크린 수직 높이)

fw = diamond_width / 2 + (diamond_width / 2 - fd)
// 또는: 좌→정점 거리 = fd, 정점→우 거리 = fw
```

#### 필수 출력

```
## 역투영 결과

### 슬래브 flat 치수
- fw (iso-X): {}px
- fd (iso-Y): {}px
- bh (body): {}px
- 비율 fw:fd = {}:1

### 콘텐츠 flat 좌표 (슬래브 #{n}: {id})
- 요소 A: translate({fx}, {fy}), 크기 {w}×{h}
- 요소 B: translate({fx}, {fy}), 크기 {w}×{h}
- 스트로크: {}px (flat space)

### 레이아웃
- Origin: ({ox}, {oy})
- iz 값: [{}, {}, {}, ...]
- iz 간격: {} iso units
```

---

### Stage 3: 구조 검증 — 수치가 말이 되는지 확인

역투영 결과를 검증한다. **하나라도 실패하면 Stage 1로 돌아간다.**

#### 검증 체크리스트

| # | 검증 항목 | 합격 기준 |
|---|---------|----------|
| 1 | fw, fd > 0 | 양수 |
| 2 | fw:fd 비율 | 1.2~3.0 범위 (직사각형) |
| 3 | 콘텐츠가 슬래브 면 안에 들어감 | 0 ≤ fx ≤ fw, 0 ≤ fy ≤ fd |
| 4 | iz 간격 균등 | 편차 < 5% |
| 5 | viewBox 내 모든 요소 포함 | margin ≥ 0 |
| 6 | 콘텐츠/슬래브 비율이 레퍼런스와 일치 | 편차 < 10% |
| 7 | 스트로크 두께가 시각적으로 충분 | flat space에서 ≥ 1px (하위 레이어), ≥ 3px (hero 콘텐츠) |

```
## 검증 결과

| # | 항목 | 값 | 기준 | 결과 |
|---|------|---|------|------|
| 1 | fw | {} | > 0 | ✅/❌ |
| 2 | fw:fd | {} | 1.2~3.0 | ✅/❌ |
| ... |
```

---

### Stage 4: 코드 생성

검증을 통과한 수치만으로 코드를 작성한다.

#### 규칙

- **모든 숫자 값 옆에 출처 주석**: `// 측정: 슬래브#4 콘텐츠 bbox에서 도출`
- **하드코딩 금지**: 상수는 Stage 2 결과에서 직접 도출
- **구조**: `isometricGrid.js` 유틸 활용 또는 로컬 빌드 함수
- **TopFaceContent**: flat 좌표계 (fw × fd) 내에서 그림
- **스트로크**: Stage 1에서 측정한 값 사용

---

### Stage 5: 시각 검증 (Playwright 비교)

생성된 SVG를 Storybook에서 렌더링하고, 레퍼런스와 프로그래밍적으로 비교한다.

#### 비교 방법

```js
// 1. 레퍼런스 이미지와 렌더링된 SVG를 동일 크기 canvas에 그린다
// 2. 주요 측정점의 좌표를 비교한다:
//    - 각 슬래브의 top vertex 위치
//    - 콘텐츠 요소의 바운딩 박스
//    - 전체 구성의 중심점
// 3. 편차가 기준치 초과 시 Stage 1로 돌아간다
```

#### 합격 기준

| 항목 | 허용 편차 |
|------|----------|
| 슬래브 위치 | ≤ 5% of viewBox |
| 콘텐츠 크기 비율 | ≤ 10% |
| 스트로크 두께 비율 | ≤ 20% |
| 전체 구성 중심 | ≤ 3% of viewBox |

---

## 안티패턴 (절대 하지 말 것)

1. ❌ "대충 이 정도면 될 것 같다" → 측정값 기반으로만 코딩
2. ❌ 스크린샷 보고 "잘 나왔다" 판단 → Playwright 측정으로 검증
3. ❌ Stage 건너뛰기 → 각 Stage 출력을 사용자에게 보여주고 승인 후 진행
4. ❌ 여러 요소를 한 번에 수정 → 한 요소씩 측정 → 코딩 → 검증
5. ❌ 시각적 추정으로 strokeWidth 결정 → 픽셀 카운트 기반

---

## 스킬 호출

사용자가 비트맵 레퍼런스를 제공하며 SVG 변환을 요청하면 이 스킬을 사용한다.
각 Stage의 출력을 사용자에게 보고하고, 승인 후 다음 Stage로 진행한다.
