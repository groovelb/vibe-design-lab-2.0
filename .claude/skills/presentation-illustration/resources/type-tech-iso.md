# Type 2: Tech Isometric — 테크 아이소메트릭

> 기존 `isometric-illustration` 스킬의 문법을 프레젠테이션 컨텍스트에 맞게 변형.
> 3D 프리미티브(Prism/Card) 기반이되, 전경을 화이트로 밝혀 슬라이드 가시성을 확보한다.

---

## 기존 아이소 스킬과의 차이

| 항목 | 기존 아이소 스킬 | 프레젠테이션 테크 아이소 |
|------|----------------|---------------------|
| **전경 밝기** | 히어로만 `--vdl-200` | 전체적으로 밝음 (`--vdl-100` 기반) |
| **fill** | `--vdl-950` (배경 동화) | `--vdl-950` 유지 (동일) |
| **캔버스** | ~270px 정사각 | 400~640px (슬라이드 스케일) |
| **strokeWidth** | 0.5 고정 | 0.5 유지 (동일) |
| **위계 단계** | 4단계 (200/600/700/800) | 3단계 (100/400/700) |
| **쉐도우** | 밀도 기반 blur | 동일 공식 적용 |

---

## 핵심 원칙

- **기존 아이소 문법을 존중한다** — Prism/Card 경로 공식, 아이소 각도 동일
- **밝기를 한 단계 올린다** — 200→100, 600→400, 700→600으로 시프트
- **슬라이드 스케일에 맞춘다** — 캔버스가 커지므로 요소 수 증가 가능

---

## 색상 규칙

| 역할 | 토큰 | 기존 대비 |
|------|------|----------|
| **면 채움** | `--vdl-950` | 동일 |
| **히어로 스트로크** | `--vdl-100` | ← `--vdl-200`에서 시프트 |
| **기본 스트로크** | `--vdl-400` | ← `--vdl-600`에서 시프트 |
| **서브틀 스트로크** | `--vdl-600` | ← `--vdl-700`에서 시프트 |
| **구조선 (V자)** | `--vdl-700` | ← `--vdl-800`에서 시프트 |
| **히어로 내부선** | `--vdl-600` | ← `--vdl-700`에서 시프트 |

### 시프트 규칙

전체적으로 **1~2단계 밝게** 시프트한다. 이유:
- 프레젠테이션은 원거리 시인성이 필요 (프로젝터, 대형 스크린)
- 기존 아이소는 모니터 근거리 감상 기준

---

## 프리미티브

기존 아이소 스킬의 Prism/Card를 그대로 사용한다.

### Prism (육각 프리즘)

```
M {cx} {topY} l {hd} {hd/2} v {bh} l {-hd} {hd/2} l {-hd} {-hd/2} v {-bh} z
V-line: M {cx-hd} {topY+hd/2} l {hd} {hd/2} l {hd} {-hd/2}
```

### Card (평행사변형)

```
M {x} {y} l {d} {d/2} v {h} l {-d} {-d/2} z
V-line: M {x+0.5} {y+2} l {d-1} {(d-1)/2} v {h-3}
```

### 아이소메트릭 각도

~26.57° (atan(1/2)) — 기존과 동일. 변경 금지.

---

## 메타포 사전

기존 아이소 스킬의 메타포 사전을 그대로 사용한다.
상세는 `.claude/skills/isometric-illustration/SKILL.md` 1단계 참조.

| 패턴 | 배치 방식 | 프리미티브 |
|------|----------|----------|
| 기반/원칙 | 수평 레이어 적층 | Prism |
| 모듈/구성 | 독립 블록 클러스터 | Prism |
| 속도/모멘텀 | 대각선 캐스케이드 | Card |
| 계층/위계 | 크기 차이 피라미드 | Prism |
| 확장/분화 | 중심에서 방사 | Prism |
| 수렴/통합 | 외곽에서 중심으로 | Prism |
| 순환/반복 | 링형 또는 순환 배치 | Card |
| 대비/전환 | 좌우 또는 전후 분할 | Prism |

## 쉐도우

기존 아이소 스킬과 동일한 공식. `visual-grammar.md` 참조.

| 인스턴스 밀도 | stdDeviation | opacity | offsetY |
|-------------|-------------|---------|---------|
| 단일 (1~3개) | 6 | 0.6 | 4 |
| 중간 (4~8개) | 8 | 1.0 | 0 |
| 고밀도 (9개+) | 16 | 1.0 | 0 |

filter ID prefix: `pi_ti_{n}`

---

## 캔버스 스케일 조정

기존 아이소는 ~270px viewBox. 프레젠테이션용은 확대한다.

| 기존 | 프레젠테이션 | 비율 |
|------|-----------|------|
| 270px | 500px | ~1.85x |

스케일 방법:
1. 기존 규칙대로 ~270px 기준으로 설계
2. viewBox를 500px 단위로 확대
3. 요소 좌표/크기를 비례 확대
4. strokeWidth는 0.5 유지 (비례 확대하지 않음 — 상대적으로 더 섬세해짐)

또는 처음부터 500px viewBox 기준으로 설계해도 무방.
핵심은 **요소 간 비례가 기존 레퍼런스와 일치**하는 것.

## SVG 구조 예시

```jsx
<svg xmlns="http://www.w3.org/2000/svg"
     viewBox="0 0 500 500" fill="none">

  {/* 뒤쪽 요소 (서브틀) */}
  <g strokeWidth="0.5" filter="url(#pi_ti_0)">
    <path fill="var(--vdl-950)" stroke="var(--vdl-600)" d="..." />
    <path stroke="var(--vdl-700)" strokeLinecap="round" d="..." />
  </g>

  {/* 기본 요소 */}
  <g strokeWidth="0.5" filter="url(#pi_ti_1)">
    <path fill="var(--vdl-950)" stroke="var(--vdl-400)" d="..." />
    <path stroke="var(--vdl-700)" strokeLinecap="round" d="..." />
  </g>

  {/* 히어로 요소 */}
  <g filter="url(#pi_ti_2)">
    <path fill="var(--vdl-950)" stroke="var(--vdl-100)" strokeWidth="0.5" d="..." />
    <path stroke="var(--vdl-700)" strokeLinecap="round" strokeWidth="0.5" d="..." />
    {/* 히어로 내부 컨텐츠 라인 */}
    <path stroke="var(--vdl-600)" strokeLinecap="round" d="..." />
  </g>

  <defs>
    <filter id="pi_ti_0" ...> ... </filter>
    <filter id="pi_ti_1" ...> ... </filter>
    <filter id="pi_ti_2" ...> ... </filter>
  </defs>
</svg>
```
