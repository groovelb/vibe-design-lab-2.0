# Type 3: Limited Color — 제한 칼라

> 면(fill)을 허용하되 배경 제외 최대 3색으로 제한.
> 3가지 유형 중 가장 표현적. 도형, 면, 그라데이션 없는 플랫 칼라.

---

## 핵심 원칙

- **3색 이내** — 배경(`--vdl-950`) 제외, 전경에 사용하는 색상은 최대 3개
- **violetGray만** — 유채색 절대 금지. 명도 차이만으로 칼라감 표현
- **면이 주역** — 모노 라인과 달리 fill이 형태를 정의한다

---

## 색상 팔레트 조합

3색을 violetGray에서 선택한다. 권장 조합:

### 팔레트 A: 하이 콘트라스트 (기본 권장)

| 역할 | 토큰 | 용도 |
|------|------|------|
| **주색** | `--vdl-50` | 가장 밝은 면. 히어로/강조 영역 |
| **중간색** | `--vdl-400` | 세컨더리 면. 보조 영역 |
| **어두운색** | `--vdl-700` | 배경과 가까운 면. 깊이/그림자 역할 |

### 팔레트 B: 소프트 톤

| 역할 | 토큰 | 용도 |
|------|------|------|
| **주색** | `--vdl-100` | 밝지만 부드러운 면 |
| **중간색** | `--vdl-500` | 중간 톤 면 |
| **어두운색** | `--vdl-800` | 배경에 거의 녹는 면 |

### 팔레트 C: 미니멀 듀오톤 (2색)

| 역할 | 토큰 | 용도 |
|------|------|------|
| **주색** | `--vdl-50` | 밝은 면 |
| **보조색** | `--vdl-600` | 어두운 면 |

### 팔레트 선택 기준

- 강한 인상, 다이나믹한 주제 → **A (하이 콘트라스트)**
- 차분한 분위기, 배경적 역할 → **B (소프트 톤)**
- 극도의 절제, 아이콘적 → **C (듀오톤)**

---

## 스트로크 규칙

| 파라미터 | 값 |
|---------|---|
| strokeWidth | 1 (기본) |
| strokeLinecap | round |
| strokeLinejoin | round |

### 스트로크 vs Fill 관계

- **면(fill)이 있는 도형**: stroke는 `none` 또는 fill보다 1~2단계 밝은 토큰
- **선만 있는 요소**: stroke = 팔레트의 주색 또는 중간색
- fill과 stroke를 동일 색상으로 사용하지 않음 (경계 소실)

---

## 메타포 사전

면을 사용하므로 모노 라인보다 풍부한 표현이 가능하다.

| 패턴 | 표현 방식 | 적합한 주제 |
|------|----------|-----------|
| **블록 구성** | 면 채움 사각형/원의 조합 | 구조, 레이아웃, 컴포넌트 |
| **레이어 스택** | 반투명 면의 겹침 (opacity X, 색상 차이로) | 깊이, 계층, 레이어 |
| **인포그래픽** | 바/원/도넛 차트 스타일 | 데이터, 비율, 비교 |
| **아이콘 세트** | 통일된 스타일의 심볼 나열 | 기능, 도구, 카테고리 |
| **마스크/실루엣** | 큰 면 + 내부 공백으로 형태 | 제품, 디바이스, UI |
| **타일/모자이크** | 작은 면의 규칙적 배열 | 다양성, 생태계, 컬렉션 |

### 메타포 적용 예시

```
"디자인 시스템 구조" → 블록 구성 — 크기가 다른 사각형 면의 조합
"데이터 분석" → 인포그래픽 — 바 차트 스타일 면 배열
"기능 소개" → 아이콘 세트 — 4~6개 심볼의 그리드
"UI 스크린" → 마스크/실루엣 — 디바이스 프레임 + 내부 UI 블록
```

## 구성 규칙

### 면의 관계

- 면끼리 겹치면 뒤쪽 면은 더 어두운 토큰 사용
- 면 사이 간격: 최소 4px (viewBox 기준)
- 면의 모서리: 직각 (VDL borderRadius 0 원칙)

### 복잡도 제한

| 캔버스 크기 | 면 개수 상한 | 비고 |
|-----------|-----------|------|
| ~200px | 5~8개 | 아이콘급 |
| ~400px | 8~15개 | 중형 일러스트 |
| ~640px | 15~25개 | 대형 일러스트 |

### 네거티브 스페이스

- 캔버스의 30~50%는 비어있어야 한다
- 면이 있으므로 모노 라인보다 여백 비율이 약간 낮아도 됨

## SVG 구조 예시

```jsx
<svg xmlns="http://www.w3.org/2000/svg"
     viewBox="0 0 480 480" fill="none">

  {/* 배경 면 (가장 어두운 색) */}
  <rect x="120" y="100" width="240" height="280"
        fill="var(--vdl-700)" />

  {/* 세컨더리 면 */}
  <rect x="140" y="120" width="200" height="120"
        fill="var(--vdl-400)" />

  {/* 히어로 면 (가장 밝은 색) */}
  <rect x="140" y="260" width="200" height="100"
        fill="var(--vdl-50)" />

  {/* 디테일 선 */}
  <line x1="160" y1="290" x2="280" y2="290"
        stroke="var(--vdl-700)" strokeWidth="1"
        strokeLinecap="round" />

  {/* 선택적 쉐도우 */}
  <defs>
    <filter id="pi_cl_0" x="116" y="96" width="248" height="296"
            colorInterpolationFilters="sRGB" filterUnits="userSpaceOnUse">
      <feDropShadow dx="0" dy="4" stdDeviation="4"
                    floodColor="var(--vdl-950)" floodOpacity="0.5" />
    </filter>
  </defs>
</svg>
```

## 쉐도우 (선택적)

제한 칼라 유형에서 쉐도우는 **선택적**이다.

- 면의 명도 차이만으로 충분한 깊이감이 있으면 생략
- 레이어 스택 패턴에서 겹침을 강조할 때 사용
- 사용 시: stdDeviation 4~6, opacity 0.3~0.5 (아이소보다 가벼움)
