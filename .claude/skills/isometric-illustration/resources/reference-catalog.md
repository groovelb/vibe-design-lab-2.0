## FIG 0.2 — Built for purpose

### 파이프라인 역추적

| 단계 | 적용 내용 |
|------|----------|
| 키워드 | "목적에 맞게 설계된 것", "원칙 기반" |
| 메타포 | **기반/원칙** — 수평 레이어 적층 |
| 프리미티브 | Prism (육각 프리즘) |
| 배치 | 1 투명 컨테이너 + 1 불투명 플랫폼 + 5 수평선 + 돔 구조 |
| 히어로 | 상단 플랫폼 (유일한 fill + `--vdl-200` 스트로크) |

### 구조 파라미터

```
viewBox: 265 x 262
인스턴스:
  - 투명 외곽: height 88.333, stroke --vdl-200
  - 상단 플랫폼: height 9.476, fill --vdl-950, stroke --vdl-200 (히어로)
  - 수평 V자선 5개: y간격 18px, stroke --vdl-700
  - 돔 4단: 동심 윤곽선, width 점진 감소, stroke --vdl-700
  - 점선 가이드 3개: dasharray 1 3, stroke --vdl-700
대칭: x=132 기준 좌우 대칭
쉐도우: stdDeviation 6, opacity 0.6, offsetY 4
```

### 핵심 테크닉

- "내부가 보이는 투명 컨테이너" — 외곽은 스트로크만, fill 없음
- 돔 구조 = 동일 곡선의 width 파라미터만 축소하며 반복
- 점선 가이드 = "측정된 설계"의 시각적 증거

## FIG 0.3 — Powered by AI agents

### 파이프라인 역추적

| 단계 | 적용 내용 |
|------|----------|
| 키워드 | "AI 에이전트", "모듈러 워크플로우" |
| 메타포 | **모듈/구성** — 독립 블록 클러스터 |
| 프리미티브 | Prism (육각 프리즘) |
| 배치 | 3컬럼 클러스터 (좌/중앙/우), staggered y |
| 히어로 | 중앙-상단 프리즘 (가장 크고, `--vdl-200` 스트로크) |

### 구조 파라미터

```
viewBox: 304 x 281
인스턴스 6개:
  중앙-상단: x=152, y=1,   h=125, stroke --vdl-200 (히어로)
  좌측-중간: x=88,  y=53,  h=73,  stroke --vdl-200
  우측-하단: x=216, y=97,  h=61,  stroke --vdl-200
  중앙-하단: x=152, y=145, h=45,  stroke --vdl-200
  좌측-하단: x=88,  y=139, h=19,  stroke --vdl-700 (비활성)
  우측-상단: x=216, y=64,  h=19,  stroke --vdl-700 (비활성)
쉐도우: stdDeviation 16, opacity 1.0
```

### 2티어 위계

```
활성 (--vdl-200): 4개 — 핵심 모듈
비활성 (--vdl-700): 2개 — 대기/스탠바이 인스턴스
```

### 핵심 테크닉

- 높이 차이 = 역할/용량의 차이 (히어로 125 vs 스탠바이 19)
- 얇은(h=19) 비활성 모듈 = "아직 활성화되지 않은 에이전트"
- 삼각형 구도: 시각적 안정감 + 계층 암시

## FIG 0.4 — Designed for speed

### 파이프라인 역추적

| 단계 | 적용 내용 |
|------|----------|
| 키워드 | "속도", "모멘텀", "고속 배송" |
| 메타포 | **속도/모멘텀** — 대각선 캐스케이드 |
| 프리미티브 | Card (평행사변형) |
| 배치 | 15개 카드, 우상→좌하 대각선 |
| 히어로 | 5번째 카드 (가장 높고 밝음, 내부 컨텐츠 라인 보유) |

### 구조 파라미터

```
viewBox: 272 x 267
카드 15개 (뒤→앞 순서):
  #1:  x=137, y=107, h=20.5,  stroke --vdl-600
  #2:  x=128, y=98,  h=34.0,  stroke --vdl-600
  #3:  x=120, y=82,  h=54.3,  stroke --vdl-600
  #4:  x=111, y=59,  h=81.4,  stroke --vdl-600
  #5:  x=103, y=16,  h=128.7, stroke --vdl-200 (히어로)
  #6:  x=94,  y=67,  h=81.4,  stroke --vdl-600
  #7:  x=86,  y=99,  h=54.3,  stroke --vdl-600
  #8:  x=77,  y=123, h=34.0,  stroke --vdl-600
  #9:  x=69,  y=141, h=20.5,  stroke --vdl-600
  #10: x=60,  y=152, h=13.8,  stroke --vdl-600
  #11: x=52,  y=160, h=10.4,  stroke --vdl-600
  #12: x=44,  y=165, h=8.7,   stroke --vdl-600
  #13: x=35,  y=171, h=7.9,   stroke --vdl-600
  #14: x=27,  y=175, h=7.9,   stroke --vdl-600
  #15: x=18,  y=179, h=7.9,   stroke --vdl-600
x offset: ~8.45px/카드
쉐도우: stdDeviation 8, opacity 1.0
```

### 높이 곡선 (비대칭 웨이브)

```
빌드업 (뒤 4장): 20.5 → 34 → 54.3 → 81.4  (급상승, 약 1.6x씩)
히어로:          128.7                        (정점)
감속 (앞 4장):   81.4 → 54.3 → 34 → 20.5    (대칭 감소)
긴 꼬리 (6장):   13.8 → 10.4 → 8.7 → 7.9³   (점근적 수렴)
```

이 곡선이 "가속 → 최고속 → 유지"를 표현한다.

### 3티어 위계

```
히어로 (--vdl-200): 1개 — 현재 활성 작업
기본 (--vdl-600): 14개 — 처리된/대기 작업
히어로 내부 (--vdl-700): 2줄 — 코드/텍스트 암시
```

### 핵심 테크닉

- 비대칭 높이 곡선 = 가속은 급격, 감속은 완만 → "한번 붙은 모멘텀"
- 긴 꼬리의 수렴값(7.9) = 최소 산출 단위가 유지됨
- 히어로만 컨텐츠 라인 보유 = "지금 작업 중"
