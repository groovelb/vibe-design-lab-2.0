# ID Naming Rules

> 슬라이드 ID 규칙과 재번호 매기기 절차.

---

## 계층 구조

```
Section (S1, S2, ...)          ← 독립 덱 단위
  └─ Chapter (Ch1, Ch2, ...)   ← 대주제
       └─ Part (S1-P-A, 2-B, ...) ← 소주제
            └─ Slide               ← 개별 화면
```

---

## 슬라이드 ID 형식

```
{partId}-{순번}
```

| 요소 | 규칙 | 예시 |
|------|------|------|
| partId | Part의 id 값 그대로 | `S1-P-A`, `2-B`, `3-F` |
| 순번 | 0부터 시작, 갭 없이 순차 | `0`, `1`, `2`, ... |
| 구분자 | 하이픈 `-` | `S1-P-A-0`, `2-B-3` |

**예시:**
- `S1-P-A-0` → S1-P-A 파트의 첫 슬라이드 (보통 ChapterTitle)
- `S1-P-A-1` → 두 번째 슬라이드
- `2-A-4` → 2-A 파트의 다섯 번째 슬라이드

---

## 재번호 매기기 절차

### 슬라이드 삽입 시

1. 삽입 위치 결정 (예: S1-P-A-2와 S1-P-A-3 사이)
2. 새 슬라이드 ID = `S1-P-A-3`
3. 기존 S1-P-A-3 → S1-P-A-4, S1-P-A-4 → S1-P-A-5, ... 순차 +1
4. grep으로 해당 Part 내 ID 중복 확인

### 슬라이드 삭제 시

1. 대상 슬라이드 제거 (예: S1-P-A-2 삭제)
2. 기존 S1-P-A-3 → S1-P-A-2, S1-P-A-4 → S1-P-A-3, ... 순차 -1
3. grep으로 해당 Part 내 ID 갭 없이 순차인지 확인

### 검증 명령

```bash
# 특정 Part의 ID 목록 확인
grep "id: 'S1-P-A-" src/data/presentations/s1.jsx
```

결과가 순차(0, 1, 2, 3, ...)이고 중복이 없어야 한다.

---

## ChapterTitle 위치 규칙

- Part의 **첫 슬라이드(순번 0)** 는 ChapterTitle
- ChapterTitle은 명시적 지시 없이 삭제하지 않는다
- 슬라이드 삽입 시 ChapterTitle 앞에 삽입하지 않는다 (항상 0번 유지)

---

## title 필드 규칙

- `title`은 ToC, Drawer, breadcrumb에 표시되는 짧은 라벨
- 슬라이드 내용을 요약하되 간결하게 (한글 기준 10~20자)
- render 내부의 실제 콘텐츠 텍스트와 다를 수 있음
