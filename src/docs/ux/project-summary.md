# Project Summary — Vibe Design Labs 웹사이트

> 이 문서는 VDL 웹사이트의 존재 이유, 핵심 가치, 범위를 정의한다.
> 이후 모든 UX·기술·콘텐츠 판단의 기준점이다.
> 작성일: 2026-02-26

---

## 한 문장 정의

**이 서비스는 디자이너를 위한 바이브 코딩 교육 플랫폼으로, 커뮤니티와 챌린지 중심의 피어 학습 환경을 통해 도구가 바뀌어도 유효한 디자인 언어 체계를 습득하게 한다.**

---

## 이 서비스가 해결하는 문제

### 1. 기존 바이브 코딩 교육의 구조적 한계

정적인 VOD 강의는 일방향이다. 혼자 듣고, 혼자 막히고, 혼자 포기한다. 완주율이 낮고, 완주해도 실전에서 써먹지 못한다. 학습자 사이의 질문, 시행착오, 결과물이 공유되지 않으니 동기부여가 끊긴다.

**VDL의 접근:** 코스 하나가 하나의 커뮤니티처럼 운영된다. 챕터별 질문과 챌린지가 실시간으로 공개되고, 멤버 간 결과물이 공유되며, 동료 압력이 학습 동력이 된다. 정적인 콘텐츠 소비가 아니라 SNS 친화적 피어 학습 경험.

### 2. 도구 종속적 학습의 허무함

v0, Cursor 사용법을 배워도 6개월 뒤 도구가 바뀌면 처음부터 다시 배워야 한다. 도구는 바뀌지만 디자인 언어 체계(패턴 분류, 토큰 구조, 시스템 사고)는 유효하다. 시장의 대부분의 교육이 도구 사용법에 집중하고, 도구 위에서 작동하는 언어를 다루지 않는다.

**VDL의 접근:** 한 번 배우면 계속 가는 디자인 언어 체계를 가르친다. 코스의 본체는 메타 지식(디자인 원리, 시스템 사고)이고, 도구와 예제 환경은 연 2회 업데이트되는 껍데기다. Vibe Dictionary가 이 체계의 산출물이자 살아있는 연구물이다.

### 3. 학습 이후 단절

코스를 끝내면 끝이다. 졸업생의 성장, 결과물, 증언이 다음 학습자에게 전달되지 않는다. 교육 → 결과 → 마케팅 → 유입의 순환이 없다.

**VDL의 접근:** 멤버가 운영과 마케팅에 연계되는 지속 가능한 생태계. 기존 멤버들의 요청에 따라 연 2회 코스 업데이트. 멤버들의 결과물과 증언이 다음 코호트의 Attract 자산이 된다. 졸업생이 커뮤니티에 남아 피어 리뷰어가 되고, 그 순환 자체가 해자다.

## 핵심 가치 — 이 플랫폼이 전달해야 하는 경험적 약속

| 가치 | 경험 |
|------|------|
| **혼자 듣는 강의가 아니라 함께 만드는 학습** | 코스가 커뮤니티처럼 운영된다. 질문이 공개되고, 챌린지 결과가 공유되고, 동료의 시행착오에서 배운다. "같이 하니까 끝까지 간다" |
| **도구가 바뀌어도 남는 것을 배운다** | 특정 도구 사용법이 아니라 디자인 언어 체계를 습득한다. Dictionary의 키워드는 v0에서도, Cursor에서도, 다음 세대 도구에서도 작동한다 |
| **"감이 아니라 설계"를 직접 체감** | 플랫폼을 탐색하면서 VDL의 정밀함과 체계를 느낀다. 사이트 자체가 VDL 품질의 증거다 |
| **멤버의 성장이 플랫폼의 성장** | 멤버의 결과물과 증언이 다음 학습자의 동기가 된다. 쓰고 버리는 강의가 아니라 순환하는 생태계 |

## 타겟 사용자

### Primary (70%)
**캔버스 디자이너** — 3~8년차 UX/UI/프로덕트 디자이너. Figma 안에서만 영향력이 있고, 구현은 남의 몫인 상태. "내 디자인을 끝까지 통제하고 싶다."

### Secondary (20%)
**파이프라인 개발자** — 2~7년차 프론트엔드/풀스택 개발자. 구현은 되는데 제품 의사결정의 언어가 없는 상태. "왜 어색한지 모르겠다."

### Tertiary (10%)
**샌드위치 PM** — 2~6년차 PM/기획자. 말로 설득해야 하는데 보여줄 게 없는 상태. "그냥 만들어서 보여주면 안 되나."

## 서비스 시나리오

### 시나리오 1: 코스 = 커뮤니티

코스 하나가 독립된 커뮤니티처럼 운영된다. 플랫폼 내 코스 페이지와 Discord가 연동되어, 챕터별 질문과 챌린지 내용이 양쪽에서 공유된다. 멤버는 혼자 영상을 보는 게 아니라, 같은 챕터를 진행 중인 동료의 질문을 읽고, 자기 결과물을 올리고, 피드백을 주고받는다.

```
코스 진행 플로우:
[챕터 학습] → [챌린지 수행] → [결과물 공유] → [피어 피드백] → [다음 챕터]
                                    ↕
                            [Discord 실시간 소통]
```

### 시나리오 2: 인터랙티브 학습 환경

정적 VOD가 아닌, 챕터별 질문과 논의가 실시간으로 공개되는 SNS 친화적 학습 UX. 각 챕터 페이지에서 멤버들의 질문·답변·결과물이 타임라인처럼 흐르고, 다른 멤버의 시행착오가 자연스럽게 학습 자료가 된다.

| 기존 VOD | VDL 플랫폼 |
|---------|-----------|
| 혼자 재생, 혼자 정지 | 챕터마다 동료의 질문과 결과물이 보인다 |
| 질문은 게시판에 묻힌다 | 질문이 챕터 맥락에 바로 연결된다 |
| 완주해도 아무도 모른다 | 챌린지 결과가 공유되고, 반응이 온다 |
| 끝나면 끝 | 졸업 후에도 커뮤니티에 남는다 |

### 시나리오 3: 멤버 → 운영 → 마케팅 순환

멤버의 학습 결과와 증언이 플랫폼 운영과 마케팅에 직접 연계되는 지속 가능한 생태계.

```
[멤버 학습]
    ↓ 결과물 + 증언 축적
[커뮤니티 자산]
    ↓ 다음 코호트 Attract
[신규 멤버 유입]
    ↓ 코스 업데이트 요청
[연 2회 코스 업데이트]
    ↓ 기존 멤버도 혜택
[멤버 충성도 강화]
    ↓ 마케팅 참여 유도
[멤버가 곧 마케터]
```

- 기존 멤버들의 요청과 피드백에 따라 연 2회 코스 콘텐츠 업데이트 (메타 지식은 유지, 도구 예제만 교체)
- 졸업생 결과물·증언이 Landing, Course 페이지의 소셜 프루프로 활용
- 멤버의 자발적 공유(SNS, 커뮤니티)가 자연스러운 마케팅 채널

## 페이지 구성과 각 페이지의 목적

### 1. Landing
**목적:** 브랜드 스토리텔링 + 다른 페이지로의 허브 + Course CTA

| 역할 | 설명 |
|------|------|
| 브랜드 스토리텔링 | VDL의 한 문장 정의, 태그라인, 가치 제안을 시각적으로 전달 |
| 섹션 허브 | Course, Dictionary, Brand Story, Brand Experiment로 연결되는 섹션 구성 |
| 소셜 프루프 | 멤버 결과물, 증언, 커뮤니티 활동 스냅샷 |
| CTA | Course 페이지로의 전환 유도가 최종 목표 |

### 2. Course
**목적:** 코스 목록 탐색 + Course Detail로의 진입

| 역할 | 설명 |
|------|------|
| 코스 리스트 | 현재 2개 코스 대기 중. 각 코스의 핵심 가치, 대상, 커뮤니티 규모를 한눈에 |
| 학습 방식 소개 | "VOD가 아닌 커뮤니티 학습"이라는 차별점을 이 페이지에서 명확히 전달 |
| 탐색 | 코스 카드에서 Course Detail로 진입 |

### 3. Course Detail
**목적:** 개별 코스의 상세 정보 + 커뮤니티 학습 경험 미리보기 + 수강 결심

| 역할 | 설명 |
|------|------|
| 코스 설명 | 커리큘럼, 대상, 기대 결과, 기간, 가격 |
| 학습 환경 미리보기 | 챕터별 질문/챌린지 피드, 멤버 결과물 미리보기 — "이런 환경에서 배운다"를 보여줌 |
| 멤버 증언 | 이전 코호트 멤버의 결과물과 증언 |
| 전환 | 수강 신청 CTA |

### 4. Vibe Dictionary
**목적:** VDL 연구 자산의 체험 — "이 체계가 진짜다"의 물적 증거

| 역할 | 설명 |
|------|------|
| Design Taxonomy | 디자인 패턴 분류 체계. 인터랙티브 트리 구조로 탐색 |
| Vibe Wiki (추가 예정) | 개별 키워드의 상세 설명, 예시, 관련 패턴 |

### 5. Brand Experiment
**목적:** VDL의 연구 결과물 시연 — "정밀하게 말하면 정밀하게 나온다"의 실물 증거

| 역할 | 설명 |
|------|------|
| 예제 갤러리 | VDL 디자인 언어로 만든 결과물 예제 (추가 예정) |
| Before/After | 동일 의도, 다른 프롬프트의 결과 차이 시연 |

### 6. Brand Story
**목적:** VDL의 정체성과 철학을 깊이 있게 전달

| 역할 | 설명 |
|------|------|
| 미션 | "사고와 구현의 주체를 일치시킨다" |
| Philosophy | 세 가지 신념 — 구현은 언어를 따른다 / 언어는 경험을 함축한다 / 감각도 쪼개면 로직이다 |
| 가치 제안 | System Over Drawing → The Vibe Standard → Design As Build |

## 제공하지 않는 것 (Non-goal)

| Non-goal | 이유 |
|----------|------|
| **자체 VOD 호스팅 / 영상 플레이어** | 영상 콘텐츠는 외부 플랫폼(유튜브 임베드 등)으로 처리. 플랫폼의 본질은 커뮤니티 학습 환경 |
| **자체 결제 시스템** | 코스 결제는 외부 결제 링크로 처리. 플랫폼 내 직접 결제 구현은 Phase 1 범위 밖 |
| **범용 SNS / 블로그 기능** | 플랫폼 내 소통은 코스 맥락에 한정. 범용 게시판, 블로그는 만들지 않는다 |
| **실시간 채팅** | 실시간 소통은 Discord에서 담당. 플랫폼은 비동기 질문·결과물 공유에 집중 |
| **다국어 지원** | Phase 1은 한국어 우선. 영문은 브랜드명, 슬로건 등 고정 문구만 |
| **AI 기반 자동 피드백** | 피드백은 프랙티셔너와 동료 멤버가 한다. AI 자동 리뷰는 VDL 철학과 맞지 않는다 |

## 성공 기준

### 정성적
- 멤버가 "혼자 듣는 강의"가 아니라 "같이 하는 경험"으로 인식한다
- 챕터별 질문 피드를 보면서 "나만 막힌 게 아니구나"를 느낀다
- 코스를 끝낸 멤버가 자발적으로 결과물을 공유하고, 그 공유가 신규 유입으로 이어진다
- "VDL이 뭐 하는 곳이야?"라는 질문에 사이트 링크 하나로 답할 수 있다
- 사이트를 본 사람이 "교육 업체"가 아니라 "연구하고 함께 만드는 곳"으로 인식한다
- Dictionary 페이지에서 "이런 체계가 있었어?"라는 반응이 나온다

### 정량적 (Phase 1 목표)
- Landing → Course 페이지 이동률
- Course → Course Detail 진입률
- Course Detail → 수강 신청 CTA 클릭률
- 코스 내 챕터별 질문·결과물 공유 비율 (멤버 참여도)
- 코호트 완주율
- 졸업생 자발적 공유 건수 (SNS, 커뮤니티)

## 브랜드 톤 원칙 (사이트에 적용)

이 사이트의 모든 카피, 시각 요소, 인터랙션은 VDL 퍼스널리티 5축을 따른다:

| 축 | 사이트에서의 표현 |
|---|---|
| **주도** | 단정적 카피. "~일 수 있습니다"가 아니라 "~입니다" |
| **정밀** | 추상적 형용사 대신 구체적 키워드와 수치 |
| **절제** | 장식 없음. 모든 시각 요소가 의미를 가진다 |
| **독립** | 타사 권위를 빌리지 않는다. 자체 증거로 말한다 |
| **본질** | 트렌드가 아니라 원리. 도구가 바뀌어도 남는 것 |

### 시각 블랙리스트 (사이트에서 절대 사용 금지)
그래디언트 배경, 블러리 글로우, 글라스모피즘, 뉴럴 네트워크/로봇 이미지, 캐릭터/마스코트, 추상적 블롭, 스톡 사진, 코스 카드 그리드(Udemy 스타일)

### 시각 화이트리스트
1px Monoline, Naming Line + 레이블, 따뜻한 다크 배경, 8px Grid, 산세리프 + 모노스페이스 공존, Before/After 구조, Isometric 해부 일러스트

## 유저 플로우 개요

```
[외부 유입]
  SNS / 검색 / 멤버 공유 / 졸업생 증언
       │
       ▼
  [Landing]  ─── 브랜드 스토리텔링 + 섹션 허브
       │
       ├──→ [Course] ──→ [Course Detail] ──→ 수강 신청 CTA
       │                       │
       │                       └──→ 학습 환경 미리보기 (질문 피드, 챌린지, 결과물)
       │
       ├──→ [Vibe Dictionary] ──→ Taxonomy 트리 탐색 ──→ (Wiki 추가 예정)
       │
       ├──→ [Brand Experiment] ──→ 예제 탐색
       │
       └──→ [Brand Story] ──→ 미션, Philosophy, 가치 제안
```

**핵심 전환 경로:** Landing → Course → Course Detail → 수강 신청
**보조 경로:** Dictionary, Brand Experiment, Brand Story는 브랜드 신뢰를 쌓고 Course 전환을 지원
**순환 경로:** 멤버 결과물·증언 → Landing 소셜 프루프 → 신규 유입 → Course 전환

## 플랫폼-Discord 연동 구조

```
[VDL 플랫폼]                              [Discord]
┌────────────────────────┐          ┌──────────────────────┐
│ Course Detail          │          │ #코스-채널            │
│  └ 챕터별 질문/결과물    │  ←연동→  │  └ 실시간 질문/논의   │
│     피드 (비동기)       │          │     (실시간)          │
│                        │          │                      │
│ 멤버 프로필             │          │ #챌린지-공유          │
│  └ 진행 상태/결과물     │          │  └ 결과물 올리기      │
│                        │          │                      │
│ 공개 피드              │          │ #프롬프트-클리닉       │
│  └ 전체 멤버 활동 타임라인│         │  └ 피어 피드백        │
└────────────────────────┘          └──────────────────────┘
```

플랫폼은 **구조화된 비동기 학습**(챕터 맥락에 연결된 질문, 결과물 아카이브)을 담당하고,
Discord는 **실시간 소통**(즉각 질문, 자유 논의, 동료 응원)을 담당한다.
