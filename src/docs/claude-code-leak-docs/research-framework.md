## 2. 리서치 방법론 프레임워크

### 6가지 분석 기법

각 미탐색 영역에 적합한 분석 기법을 조합하여 적용.

```
┌─────────────────────────────────────────────────────────────┐
│                    리서치 방법론 6종                          │
├─────────────┬───────────────────────────────────────────────┤
│ M1. 코드 정독  │ 핵심 파일을 순차적으로 읽고 로직 흐름 추적      │
│ (Deep Read)  │ - 대상: 100줄 이상의 핵심 모듈                  │
│              │ - 산출: 로직 플로우 요약, 핵심 함수 목록          │
├─────────────┼───────────────────────────────────────────────┤
│ M2. 패턴 마이닝 │ grep/ripgrep로 특정 패턴을 코드 전체에서 추출   │
│ (Pattern Mine)│ - 대상: 키워드, 함수명, 상수, 에러 코드          │
│              │ - 산출: 출현 위치 + 빈도 + 컨텍스트 맵           │
├─────────────┼───────────────────────────────────────────────┤
│ M3. 의존성 추적 │ import/require 체인을 따라 모듈 관계 추적       │
│ (Dep Trace)  │ - 대상: 핵심 모듈의 import 트리                 │
│              │ - 산출: 의존성 그래프 (노드/엣지 데이터)          │
├─────────────┼───────────────────────────────────────────────┤
│ M4. 데이터 흐름 │ 데이터가 생성→변환→소비되는 경로 추적           │
│ (Data Flow)  │ - 대상: 이벤트, 메트릭, 설정값, 상태 객체        │
│              │ - 산출: 데이터 플로우 다이어그램 데이터            │
├─────────────┼───────────────────────────────────────────────┤
│ M5. 프로토콜 추출│ 메시지 포맷, API 스키마, 통신 규약 추출         │
│ (Protocol)   │ - 대상: 에이전트 통신, API 엔드포인트, 이벤트 타입 │
│              │ - 산출: 프로토콜 명세서                          │
├─────────────┼───────────────────────────────────────────────┤
│ M6. 프롬프트 고고학│ 시스템 프롬프트 원문 발굴 및 의도 분석        │
│ (Prompt Arch) │ - 대상: prompts.ts, coordinatorMode.ts 등     │
│              │ - 산출: 프롬프트 구조 분해, 행동 지시 목록        │
└─────────────┴───────────────────────────────────────────────┘
```

### 적용 원칙

1. **한 영역에 복수 기법 조합** — 예: autoDream은 M1(코드 정독) + M4(데이터 흐름) + M6(프롬프트 고고학)
2. **깊이보다 연결** — 개별 파일의 모든 줄을 읽기보다, 노드 간 관계를 밝히는 데 집중
3. **시각화 우선** — 분석 결과를 항상 노드/엣지 데이터로 변환 가능한 형태로 기록
4. **레이어 교차 추적** — L2 엔진이 L3 텔레메트리에 어떻게 데이터를 먹이는지 등 계층 간 연결 우선
