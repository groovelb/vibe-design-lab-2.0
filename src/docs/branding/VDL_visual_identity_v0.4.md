# VDL 비쥬얼 브랜딩 아이덴티티 v0.4

> **상위 문서:** VDL 버벌 브랜드 아이덴티티 v1.0 · VDL 비쥬얼 전략 v1.0 · 세계관 비쥬얼 번역 비교분석 v1.2
> **목적:** 키비쥬얼 디벨롭 + 브랜드 시각 자산의 원칙과 스타일 정립
> **범위:** 브랜드 비쥬얼 아이덴티티. UI 디자인 시스템(컴포넌트, 레이아웃)은 별도 문서.
> **주요 레퍼런스:** Linear.app (monoline + dark UI + 구조적 미니멀리즘)

---

## 1. 설계 선언

**감각도 쪼개면 로직이다 — 이 문서가 그 증거다.**

VDL의 비쥬얼 아이덴티티는 이 철학의 실행 증거다. 시각 결정에 "감"으로 정한 값은 없고, 모든 값에 근거가 있으며, 코드로 재현 가능한 구조를 지향한다. 로고를 손으로 그리지 않고 파라미터로 정의한다.

**핵심 원칙:**

- 모든 시각 자산은 파라미터로 정의 가능해야 한다
- 베리에이션은 "새로 그리는" 것이 아니라 규칙 변환으로 파생한다
- 이 문서의 값만으로 시각 자산을 재구성할 수 있어야 한다

## 2. 비쥬얼 원칙

| # | 원칙 | 영향 |
| --- | --- | --- |
| 0 | **감각도 쪼개면 로직이다** | 모든 시각 결정이 파라미터로 정의된다 |
| 1 | **제거가 크래프트다** | 장식 요소 0. 모든 시각 요소가 의미를 가진다 |
| 2 | **소재의 온도가 차별화다** | 차갑게 느껴지면 틀렸다. 무채색이되 무감정은 아니다 |
| 3 | **체계가 그래픽이다** | 구조(토큰, 키워드, 코드) 자체가 그래픽 역할 |
| 4 | **보이지 않는 것을 보이게** | 규칙·토큰·관계를 가시화한다 |
| 5 | **문서가 제품이다** | 시각적 품질 = 브랜드 증거 |

### 2.1 시각 블랙리스트

| 금지 요소 | 이유 |
| --- | --- |
| 그래디언트 배경 / 블러 글로우 | AI 클리셰 |
| 글라스모피즘 / 그래디언트 헤딩 | 장식 |
| 뉴럴 네트워크 / 로봇 이미지 | AI 클리셰 |
| 캐릭터 / 마스코트 / 사실적 일러스트 | 교육 업체 클리셰 |
| 추상적 블롭 / 유기적 형태 | 파라미터 정의 불가 |
| 스톡 사진 / 장식적 3D | 원칙 1, 5 위반 |

## 3. 컬러

### 3.1 컬러 스킴

**기조: Violet-tinted Grayscale + 무채색 Primary**

Linear.app처럼 순수 gray가 아닌 미세하게 violet으로 tilt된 grayscale을 사용한다. Primary color는 블랙 & 화이트 — 유채색 accent 없이 명도 대비만으로 위계를 만든다. 다크 모드 기본.

| 역할 | 토큰명 | 방향 | 비고 |
| --- | --- | --- | --- |
| **배경** | `bg-primary` | 거의 검정에 가까운 violet-gray | 순수 검정(#000) 아님. HSL hue 250–270° 부근, saturation 3–8% |
| **배경 보조** | `bg-secondary` | 배경보다 한 단계 밝은 violet-gray | 카드, 코드 블록, 섹션 분리 |
| **배경 3차** | `bg-tertiary` | bg-secondary보다 한 단계 밝음 | 호버, 선택 상태 배경 |
| **텍스트 기본** | `text-primary` | 거의 흰색에 가까운 violet-gray | 순백(#FFF) 아님. 미세한 violet tint 유지 |
| **텍스트 보조** | `text-secondary` | 중간 밝기 violet-gray | 보조 설명, 캡션, 비활성 |
| **텍스트 3차** | `text-tertiary` | text-secondary보다 어두움 | 플레이스홀더, 메타 정보 |
| **Monoline / Grid** | `line-subtle` | 배경보다 약간 밝은 violet-gray | Grid 배경, 비활성 선, 디바이더 |
| **활성 선** | `line-active` | text-primary와 동일하거나 근접 | 로고, Naming Line, 아이콘 공통 |
| **코드 구문** | `syntax-*` | 유채색 | 코드 하이라이팅에서만 유채색 허용 |

**규칙:**

- 다크 모드 기본. 라이트 모드는 후순위.
- Primary color는 블랙 & 화이트. 유채색 accent 없음.
- 시각 위계는 색상이 아닌 **명도 차이**로만 구성한다. 밝기 단계(bg → line-subtle → text-tertiary → text-secondary → text-primary)가 정보 위계와 일치해야 한다.
- 모든 gray에 동일한 violet hue(250–270°)를 미세하게 유지. 순수 neutral gray(saturation 0%) 금지.
- 유채색은 코드 구문 하이라이팅에서만 허용.
- 모든 컬러는 토큰으로 정의. 하드코딩 금지.
- hex/HSL 값은 키비쥬얼 프로토타이핑과 함께 확정 예정.

### 3.2 컬러 적용 규칙

| 맥락 | 배경 | 텍스트 | 선 |
| --- | --- | --- | --- |
| 웹/앱 기본 | `bg-primary` | `text-primary` | `line-subtle` |
| 카드/섹션 | `bg-secondary` | `text-primary` | `line-subtle` |
| 호버/선택 | `bg-tertiary` | `text-primary` | `line-active` |
| SNS 콘텐츠 | `bg-primary` | `text-primary` | `line-active` |
| 코드 블록 | `bg-secondary` | `syntax-*` | — |
| 비활성/보조 | — | `text-tertiary` | `line-subtle` |

## 4. 서체

### 4.1 서체 체계

**산세리프 + 모노스페이스 공존 = VDL 정체성.**

디자인 언어(산세리프)와 코드 언어(모노스페이스)가 같은 화면에 공존하는 것이 VDL의 "디자인 키워드 ↔ 토큰 ↔ 코드" 등가 구조를 시각적으로 표현한다.

| 용도 | 분류 | 웨이트 | 사용처 |
| --- | --- | --- | --- |
| **Display** | 산세리프 | ExtraBold (800) | 히어로 헤딩, 선언적 카피, SNS 타이틀 |
| **Heading** | 산세리프 | SemiBold (600) | 섹션 제목, 카드 타이틀 |
| **Body** | 산세리프 | Regular (400) | 본문 |
| **Caption** | 산세리프 | Light (300) | 보조 텍스트, 메타 정보 |
| **Code / Keyword** | 모노스페이스 | Medium (500) | 디자인 키워드, 토큰명, 인라인 코드 |
| **Code Block** | 모노스페이스 | Regular (400) | 코드 블록, 터미널 |

### 4.2 서체 규칙

- 서체명은 렌더링 테스트 후 확정. 한글 + 영문 조합 고려.
- Display는 짧은 카피에만. 본문에 ExtraBold 사용 금지.
- 키워드를 언급할 때는 항상 모노스페이스(`Code/Keyword`)를 사용해 일반 텍스트와 구분.
- Linear.app 참조: 산세리프 + 모노스페이스 코드 조합의 시각적 균형.

### 4.3 타이포 스케일

| 레벨 | 사이즈 | 행간 | 용도 |
| --- | --- | --- | --- |
| Display | 48–64px | 1.1 | 히어로 |
| H1 | 32–40px | 1.2 | 페이지 타이틀 |
| H2 | 24–28px | 1.3 | 섹션 타이틀 |
| H3 | 18–20px | 1.4 | 서브섹션 |
| Body | 16px | 1.6 | 본문 |
| Caption | 14px | 1.4 | 보조 텍스트 |
| Code Inline | 14px | — | 인라인 코드/키워드 |
| Code Block | 14px | 1.5 | 코드 블록 |

## 5. 스페이싱 & 그리드

### 5.1 기본 단위

**Base unit: 8px**

모든 스페이싱, 패딩, 마진은 8px의 배수를 사용한다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `space-xs` | 4px | 최소 간격 (예외적 사용) |
| `space-sm` | 8px | 아이콘-레이블 간격, Naming Line label-gap |
| `space-md` | 16px | 요소 간 기본 간격 |
| `space-lg` | 24px | 섹션 내 블록 간 간격 |
| `space-xl` | 32px | 섹션 간 간격 |
| `space-2xl` | 48px | 대블록 간 간격 |
| `space-3xl` | 64px | 히어로/섹션 상하 여백 |

### 5.2 Grid 배경 패턴

| 파라미터 | 값 |
| --- | --- |
| grid-unit | 8px |
| grid-color | `line-subtle` 토큰, opacity 한 단계 더 낮춤 |
| grid-style | dot 또는 line (맥락에 따라 선택) |
| grid-weight | 1px |

Grid 배경은 VDL 콘텐츠의 기본 무대. Linear.app의 dot grid를 참조하되, 콘텐츠 가독성을 해치지 않는 수준으로 억제한다.

### 5.3 Monoline 공통 파라미터

로고, 아이콘, Naming Line, 일러스트가 공유하는 선 규격:

| 파라미터 | 값 | 근거 |
| --- | --- | --- |
| stroke-weight | 1px | 시스템 전체 일관성 |
| stroke-cap | round | 기하학적이되 날카롭지 않게 (원칙 2) |
| stroke-join | round | stroke-cap과 통일 |

## 6. 그래픽 모티프

### 6.1 Naming Line — 최소 패턴 단위

VDL 비쥬얼의 핵심 단위. "보이는 것에 이름을 붙이는 행위"의 시각화.

| 파라미터 | 값 |
| --- | --- |
| dot-size | 4px |
| line-weight | 1px (monoline 공통) |
| line-direction | 직각 꺾임만. 대각선 없음 |
| label-font | 모노스페이스 (Code/Keyword) |
| label-size | 14px |
| label-gap | 8px (`space-sm`) |
| multi-gap | 16px (`space-md`) |

### 6.2 Circuitry 연결선

Naming Line의 확장. 복수 요소 연결 시 사용. 1px monoline, 직각 꺾임, 대각선 없음. Linear.app의 feature connection line 문법 참조.

### 6.3 Border Highlight

해부 대상 윤곽 강조. Naming Line이 가리키는 대상을 시각적으로 분리하는 미세한 하이라이트.

## 7. 일러스트 & 아이콘 방향

### 7.1 브랜드 일러스트

**알고리즘과 데이터로만 그릴 수 있는 아이소메트릭 + 제너레이티브 일러스트.**

- 30° 아이소메트릭 투영. UI 컴포넌트를 z축 분해해 레이어 구조를 노출
- Naming Line 부착으로 각 요소에 이름/값 표기
- 손으로 그린 일러스트 없음. 모든 형태는 좌표, 각도, 반복 규칙으로 재현 가능
- 레퍼런스: Linear.app의 monoline feature 일러스트, MIT Media Lab(2011) 알고리즘 로고 베리에이션, Cohere(Pentagram) Voronoi 패턴 생성

### 7.2 키 일러스트 목록

| 일러스트 | 내용 | 우선순위 |
| --- | --- | --- |
| **키비쥬얼 대표 장면** | UI 해부 + Naming Line + Isometric 분해 | 높음 |
| **Philosophy 가치 3종** | ① 입력↔결과 대응 ② 함축→전개 ③ 결과물 분해 | 중간 |
| **Before/After** | 이름 없는 프롬프트 vs 이름 있는 프롬프트 결과 대비 | 높음 |
| **등가 다이어그램** | [디자인 키워드] — [토큰] — [코드] 3컬럼 연결 | 중간 |
| **코스 모듈 커버** | 모듈별 해부 대상 복잡도 점진적 상승 | 낮음 |

### 7.3 아이콘 시스템

로고와 동일한 DNA를 공유하는 아이콘 세트.

| 파라미터 | 값 |
| --- | --- |
| canvas | 24 × 24px |
| padding | 2px |
| stroke | 1px, round cap/join (= 로고 동일) |
| fill | none (outline only) |
| color | `line-active` 토큰 (= 로고 동일) |

카테고리: 네비게이션, 콘텐츠 유형, 상태, 도구, 브랜드 고유(Naming Line, 분해, 등가 아이콘).

## 9. 시각 계층 L1 / L2 / L3
