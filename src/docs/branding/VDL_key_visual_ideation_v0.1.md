# VDL 키 비주얼 아이데이션 v0.1

> **상위 문서:** VDL 비주얼 아이덴티티 v0.4 · VDL 버벌 브랜드 아이덴티티 v1.0 · VDL 브랜딩 전략 v2.1
> **목적:** 키 비주얼의 세 가지 방향성을 레퍼런스와 함께 아이데이션하고, 프로토타이핑 이전 단계의 시각적 가설을 정리한다
> **범위:** 로고, 키 비주얼, 그래픽 시스템의 방향성 탐색. 확정이 아닌 아이데이션.

---

## 0. 세 가지 방향성

VDL의 키 비주얼은 세 방향이 하나의 시각 체계로 수렴해야 한다.

```
[1] Generative Visual ─── 모든 시각 요소는 알고리즘과 데이터로 만든다
        │
        ├── "감각도 쪼개면 로직이다"의 시각적 증거
        │
[2] Code is Beautiful ─── 말이 코드가 되고, 코드가 디자인이 된다
        │
        ├── HTML의 <를 VDL의 V로 형상화
        ├── 코드에 대한 두려움을 제거하는 시각 언어
        │
[3] Materialize ────────── 그래픽 모티프를 물리적으로 표현한다
        │
        └── Naming Line, Circuitry, Grid를 만질 수 있는 것으로
```

세 방향은 독립이 아니다. **제너레이티브 시스템이 코드로 구동되고, 그 코드가 아름다우며, 그 아름다움이 물리적으로 만져지는 것** — 이 연쇄가 VDL 키 비주얼의 핵심 구조다.

---

## 1. Generative Visual — 모든 시각 요소는 바이브 디자인이다

### 1.1 선언

VDL은 "감각도 쪼개면 로직이다"를 말하는 브랜드다. 그렇다면 VDL의 시각 자산 자체가 그 증거여야 한다. 로고를 손으로 그리지 않고 파라미터로 정의한다(비주얼 아이덴티티 v0.4, §1). 키 비주얼도 마찬가지다 — 디자이너가 그린 한 장의 이미지가 아니라, **규칙이 만들어낸 무한한 변주**가 키 비주얼이다.

### 1.2 핵심 질문

- 어떤 파라미터가 시각 변주를 만드는가?
- 그 파라미터는 VDL의 콘텐츠/데이터와 연결 가능한가?
- "디자이너가 규칙을 만들고, 출력은 무한하되 항상 VDL로 인식되는" 시스템을 어떻게 설계하는가?

### 1.3 레퍼런스 분석

#### REF 1-A. MIT Media Lab (2011) — TheGreenEyl + E Roon Kang

**시스템:** Processing으로 제작. 4×4 그리드 위에 3개의 스포트라이트/콘 형태를 배치. 각 형태는 그리드 경계에 닿아야 하고, 최소 1개의 다른 형태와 교차해야 한다. 교차 영역이 고유한 색상 겹침을 생성.

**파라미터:** 3개 형태의 위치(그리드 내 자유 배치) × 12개 컬러 팔레트 = ~480,000개 고유 변형. 교수, 학생, 직원 각각이 고유한 로고 변형을 보유.

**VDL 적용 가능성:**
- "제약이 단순할수록 출력 공간이 넓다"는 원칙. VDL의 파라미터도 복잡한 수학이 아니라 잘 선택된 제약 조건이면 충분하다.
- MIT Media Lab의 "교차"가 개인의 만남을 시각화했듯, VDL의 제너레이티브 시스템은 **"디자인 키워드 → 토큰 → 코드"의 등가 구조**를 시각화할 수 있다.
- 코호트 참여자 각각이 고유한 비주얼 변형을 갖는 시나리오 — 졸업 후 자기 포트폴리오에 VDL 마크로 사용.

#### REF 1-B. Cohere by Pentagram — Voronoi 패턴 제너레이티브 시스템

**시스템:** 보로노이 다이어그램(공간을 시드 포인트 기반으로 분할하는 수학적 개념)을 시각 언어로 사용. Pentagram이 Cohere 디자인팀을 위해 **맞춤형 디지털 도구 모음**을 제작:
1. Figma 플러그인 — 보로노이 패턴을 Figma에서 직접 생성
2. 레이아웃 컴포넌트 라이브러리 2종 — 컨테이너 내 셀 생성 + 텍스트 주위 셀 생성
3. Cinema 4D 커스텀 도구 (Python) — 3D 조약돌 비주얼 생성. 3D 지식 불요.

**VDL 적용 가능성:**
- **"디자인 도구킷이 곧 아이덴티티"** 패턴. 가장 완성도 높은 사례. 브랜드가 단일 로고가 아니라 누구나 사용해 브랜드 일관성 있는 자산을 생산하는 제너레이티브 도구 시스템.
- VDL의 맥락: 스타터 킷 자체가 이미 코드 기반 디자인 도구. 이 도구가 VDL의 시각 자산까지 생산하는 구조 — **제품이 곧 브랜드 증거**.

#### REF 1-C. Linear.app — 코드 네이티브 비주얼 시스템

**시스템:**
1. LCH 색공간 기반 테마 생성 — 3개 파라미터(베이스 컬러, 액센트, 대비)에서 98+ CSS 변수 자동 생성
2. WebGL 메시 그래디언트 애니메이션 — Fractal Brownian Motion(심플렉스 노이즈 옥타브 레이어링)
3. 모노라인 일러스트 — 일관된 스트로크 웨이트의 기하학적 일러스트

**VDL 적용 가능성:**
- VDL의 1차 레퍼런스(비주얼 아이덴티티 v0.4, §6-7). 이미 아이소메트릭 + 모노라인 방향이 확정됨.
- Linear의 LCH 테마 시스템처럼, VDL의 violet-tinted grayscale도 **최소 파라미터에서 전체 팔레트를 생성하는 시스템**으로 설계 가능.

#### REF 1-D. Visit Nordkyn by NEUE — 실시간 데이터 기반 아이덴티티

**시스템:** 노르웨이 기상청의 실시간 기상 데이터(5분 간격 폴링)로 로고가 변형. 풍향 → 각도/형태, 온도 → 색상, 풍속 → 왜곡.

**VDL 적용 가능성:**
- "데이터가 디자인 입력"인 가장 순수한 사례. VDL에서 입력 데이터 후보:
  - Dictionary 키워드 사용 빈도
  - 코호트 참여자의 프롬프트 정밀도 점수
  - 커뮤니티 활동 데이터
- 실시간 변형까지는 아니더라도, **콘텐츠 데이터가 시각 자산의 파라미터**가 되는 구조.

#### REF 1-E. Corpus IT School by Voskhod — 텍스트 → 셀룰러 오토마타

**시스템:** 입력 텍스트를 셀룰러 오토마타 규칙으로 변환. 학교 이름, 코스명, 학생 이름이 고유한 비주얼 패턴을 생성. Conway's Game of Life에서 확장된 3D 오토마타.

**VDL 적용 가능성:**
- **입력이 언어/텍스트 자체**라는 점에서 VDL과 가장 직접적으로 연결. "디자인 키워드가 시각을 만든다"는 VDL 명제의 문자 그대로의 시각화.
- 교육 기관의 제너레이티브 아이덴티티 사례.

### 1.4 VDL 제너레이티브 시스템 가설

| 구성 요소 | 가설 | 근거 |
|----------|------|------|
| **입력 데이터** | 디자인 키워드 (Dictionary 어휘) | VDL의 핵심 자산이 시각의 입력이 된다 = 제품이 곧 브랜드 |
| **변환 규칙** | 키워드 → 아이소메트릭 프리미티브 배치 규칙 | 비주얼 아이덴티티 v0.4 §7.1의 30° 아이소메트릭 + 좌표/각도/반복 규칙 |
| **고정 요소** | 모노라인 1px, violet-tinted grayscale, 아이소메트릭 각도 | 브랜드 인식의 일관성 |
| **변동 요소** | 프리미티브 배치, Naming Line 위치, 연결선 패턴 | 무한 변주 |
| **출력** | 키워드 조합마다 고유한 아이소메트릭 장면 | 코호트별, 콘텐츠별, 사용자별 고유 비주얼 |

**핵심 원칙:** 디자이너가 규칙을 만들고, 출력은 무한하되, 항상 VDL로 인식된다.

## 2. Code is Beautiful — 말 > 코드 > 디자인

### 2.1 선언

VDL의 전체 명제를 한 문장으로: **"정밀하게 말하면 정밀하게 나온다."** 이 흐름은 시각적으로 세 단계다:

```
말 (자연어 의도)  →  코드 (구조화된 지시)  →  디자인 (시각적 결과)
     words              code                  design
```

이 흐름 자체가 VDL의 키 비주얼이다. 코드는 두려운 것이 아니라 **말과 디자인 사이의 다리**이며, 그 다리가 아름답다는 것을 보여주는 시각 체계.

### 2.2 HTML `<`를 VDL의 `V`로

#### 형상적 연결

```
HTML 태그의 여는 괄호    <
VDL의 이니셜             V
방향의 메타포             > (말에서 코드로, 코드에서 디자인으로)
```

`<`는 "여기서 인간의 의도가 기계로 들어간다"는 보편적 기호다. 동시에 방향 화살표이기도 하다. `<`를 뒤집으면 `>` — 변환과 흐름을 암시한다.

#### 로고 아이데이션

| 방향 | 설명 | 장점 | 리스크 |
|------|------|------|--------|
| **A. `<V>`** | HTML 태그 문법 안에 V를 넣는다. 코드 자체가 로고. | 기술적 정체성 명확. 코드를 아는 사람에게 즉각 공명 | 비개발자에게 낯설 수 있음 |
| **B. 기울어진 `<`** | 앵글 브라켓을 monoline으로 그려 V 형태로 만든다. | 코드와 V의 이중 읽기. 미니멀 | 너무 추상적일 수 있음 |
| **C. `< >`를 흐름으로** | 여는 `<`와 닫는 `>`를 연결선으로 이어 흐름을 시각화 | 말→코드→디자인의 방향성 표현 | 복잡해질 수 있음 |
| **D. Naming Line 부착형** | V 마크에 Naming Line을 붙여 `vibe-design-labs`를 모노스페이스로 레이블링 | VDL 그래픽 모티프(§6)와 일관. "이름 붙이기" 행위의 자기 참조 | |

#### `<` 형상의 확장 가능성

```
로고:        <V>
태그라인:    <V> Vibe Design Labs
코드 블록:   <V design="system-over-drawing" standard="vibe" build="true" />
SNS 마크:    <V>
파비콘:      <
```

단일 기호 `<`가 로고, 코드 구문, 방향 기호, 프레임으로 확장되는 체계.

### 2.3 말 > 코드 > 디자인 — 흐름의 시각화

#### REF 2-A. Bret Victor — "Inventing on Principle" (2012)

**핵심:** 코드(왼쪽 패널)가 실시간으로 시각적 출력(오른쪽 패널)을 생성. 쓰는 행위와 보는 행위 사이의 시간 간극 제거.

**VDL 적용:** VDL의 "Design at the Speed of Thought"의 시각적 원형. 키 비주얼에서 코드와 결과가 **동시에 보이는** 레이아웃 — 왼쪽에 프롬프트/코드, 오른쪽에 렌더된 디자인, 사이에 변환의 흔적(Naming Line, Circuitry).

#### REF 2-B. v0.dev (Vercel) — Words → Code → Design 파이프라인

**핵심:** 자연어로 UI를 설명 → React/Tailwind 코드 생성 → 시각적 미리보기 렌더링. 세 단계가 인터페이스에서 문자 그대로 보인다: 프롬프트(왼쪽), 코드(중간), 프리뷰(오른쪽).

**VDL 적용:** VDL이 가르치는 것의 직접적 시각화. 키 비주얼의 3단 구조:

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   WORDS     │ →  │    CODE     │ →  │   DESIGN    │
│             │    │             │    │             │
│ "Bento Grid │    │ <Grid       │    │ ┌──┬───────┐│
│  + semantic │    │   container │    │ │  │       ││
│  color      │    │   spacing=  │    │ ├──┼──┬────┤│
│  token"     │    │   {2}>      │    │ │  │  │    ││
│             │    │   ...       │    │ └──┴──┴────┘│
└─────────────┘    └─────────────┘    └─────────────┘
```

#### REF 2-C. Stripe — 코드가 브랜드다

**핵심:** 3-컬럼 레이아웃(설명 / 코드 예시 / 라이브 응답). 커스텀 구문 하이라이팅이 브랜드 컬러와 일치. 모든 코드 블록이 실행 가능. 코드가 제품이므로 코드가 브랜드.

**VDL 적용:**
- VDL도 코드를 사과할 대상이 아니라 **자랑할 소재**로 다룬다.
- 키 비주얼에서 코드 블록이 장식이 아니라 **작동하는 설계 명세**로 등장.
- Stripe가 API 호출을 애니메이션으로 보여주듯, VDL은 **프롬프트 → 디자인 변환**을 애니메이션으로 보여줄 수 있다.

### 2.4 코드의 두려움을 제거하는 시각 전략

#### REF 2-D. Scratch (MIT Media Lab) — 구문 오류 불가능한 시각 체계

**핵심:** 퍼즐 조각 형태가 코드 구문을 물리적/촉각적으로 만든다. 맞지 않는 조각은 끼울 수 없어 구문 오류가 원천 차단. 색상 코딩(동작=파랑, 외관=보라, 소리=분홍)이 논리적이 아닌 감정적 분류를 만든다.

**VDL 적용:** VDL의 디자인 키워드가 Scratch의 퍼즐 조각 역할을 한다. 키워드를 조합하면 올바른 프롬프트가 되고, 올바른 프롬프트는 의도한 디자인을 만든다. 시각적으로: Dictionary 키워드가 **조합 가능한 시각 블록**으로 표현되는 장면.

#### REF 2-E. p5.js — 코드는 창작 매체다

**핵심:** 소프트 핑크와 플레이풀한 타이포그래피. 다크 터미널 해커 미학을 의식적으로 거부. 코드 예시를 밝은 테마에 넉넉한 여백으로 보여준다. 메시지: 코드는 페인트나 클레이처럼 창작 매체다.

**VDL 적용:** VDL의 다크 모드 기조와 충돌하지 않으면서도, 코드를 "무서운 것"이 아닌 "아름다운 재료"로 다루는 톤. 핵심은 **코드 블록의 타이포그래피적 품질** — 구문 하이라이팅, 행간, 여백을 시 조판처럼 다룬다.

### 2.5 시각 블랙리스트 / 화이트리스트

| 금지 | 이유 | 대안 |
|------|------|------|
| Matrix 스타일 떨어지는 초록 코드 | 해커 클리셰. 두려움 유발 | 정적이고 읽을 수 있는 코드 블록 |
| 빽빽한 모노스페이스 텍스트 벽 | 협박적 | 넉넉한 행간, 구문 하이라이팅, 핵심 라인만 |
| `$` 프롬프트, 블링킹 커서 우선 | "이미 알아야 한다"는 신호 | 코드 결과가 먼저, 소스는 두 번째 |
| 코드를 장식으로만 사용 | 의미 없는 배경 노이즈 | 코드가 실제로 작동하는 설계 명세 |

| 권장 | 이유 |
|------|------|
| 코드 + 결과 동시 표시 | 변환의 즉각성. "이게 저걸 만든다" |
| 3-5줄의 핵심 코드만 | 압도하지 않으면서 정밀함을 보여줌 |
| 구문 하이라이팅 = 유일한 유채색 | 비주얼 아이덴티티 v0.4 §3.1: 유채색은 코드 구문에서만 허용 |
| 코드 내 VDL 키워드 볼드/하이라이트 | "디자인 언어가 코드 안에 있다" |

### 2.6 `<V>` 형상 체계 — 종합 가설

```
                    < V >
                   ╱     ╲
              여는 괄호   닫는 괄호
              입력의 시작   출력의 완결
              말의 진입     디자인의 출현
                   │
              V = Vibe Design
              V = 변환의 꼭짓점
              V = 수렴점 (말과 코드와 디자인이 만나는 곳)
```

`<V>`는 단순한 로고가 아니라 **VDL의 전체 명제를 축약한 기호**다:
- `<` = 의도의 진입 (말)
- `V` = 변환의 순간 (바이브 디자인)
- `>` = 결과의 출현 (디자인)

## 3. Materialize — 그래픽 모티프의 물리적 표현

### 3.1 선언

비주얼 아이덴티티 v0.4 §6의 그래픽 모티프(Naming Line, Circuitry, Border Highlight)는 스크린 위의 패턴이다. 키 비주얼의 세 번째 방향은 이것을 **만질 수 있는 것으로** 만드는 것이다. 디지털 모티프가 물리적 존재감을 가질 때, "보이지 않는 것을 보이게"(원칙 4)라는 VDL 선언이 문자 그대로 실현된다.

### 3.2 물리화할 모티프 목록

| 모티프 | 디지털 정의 (v0.4) | 물리적 메타포 |
|--------|-------------------|-------------|
| **Naming Line** | dot(4px) + 1px line + 모노스페이스 레이블 | 와이어 + 핀 + 엠보싱 레이블 |
| **Circuitry** | 1px monoline 직각 꺾임 연결선 | 벤딩 와이어, PCB 트레이스 |
| **Border Highlight** | 대상 윤곽 강조 | 컷아웃, 다이컷, 음각 |
| **Grid 배경** | 8px 도트/라인 그리드 | 모눈종이, 레이저 각인 그리드 |
| **아이소메트릭 분해** | 30° 투영 z축 분해 | 아크릴 레이어 분리, 와이어프레임 조각 |

### 3.3 레퍼런스 분석

#### REF 3-A. Google Material Design — 리터럴 머티리얼 메타포

**핵심:** "Quantum Paper"라는 코드명에서 시작. 디자인팀이 **물리적 프로토타입을 먼저 제작**. 실제 종이를 오리고 조명을 비춰 그림자 효과를 관찰한 뒤, 디지털 시스템을 역으로 구축. 디지털 시스템이 물리적 실험에서 파생됨.

**VDL 적용:**
- VDL도 역방향이 가능하다: 디지털 모티프(Naming Line, Circuitry)를 물리적으로 먼저 만들고, 그 물성이 디지털 시스템의 디테일을 결정.
- "종이의 물성"이 Material Design의 그림자와 겹침 규칙을 만들었듯, VDL의 물리적 메타포가 모노라인의 두께감, 연결선의 방향 전환 반경 같은 디테일을 결정할 수 있다.

#### REF 3-B. Olafur Eliasson — The Weather Project (Tate Modern, 2003)

**핵심:** 보이지 않는 현상(날씨, 빛의 스펙트럼)을 거대한 물리 설치로 가시화. 결정적으로, **메커니즘을 숨기지 않는다**. 구조와 기계가 의도적으로 노출되어, 관객이 "구성을 의식"하고, "지각 행위 자체를 의식"하게 만든다.

**VDL 적용:**
- **"보이지 않는 것을 보이게"**(원칙 4)의 가장 직접적인 물리적 레퍼런스.
- VDL 키 비주얼의 핵심: 디자인 결정의 메커니즘을 숨기지 않고 노출한다. Naming Line이 바로 그 장치 — 결과물뿐 아니라 그 결과물이 왜 그런 형태인지를 함께 보여준다.
- "메커니즘 노출 = 미학"이라는 원칙. VDL에서 파라미터 노출, 토큰명 노출, 코드 구문 노출이 미학적 선택이 되는 근거.

#### REF 3-C. iFixit / Xreart — X-ray / Teardown 미학

**핵심:** Apple 제품의 분해 사진과 X-ray 촬영을 소비자 월페이퍼로 배포. Xreart는 실제 분해된 디바이스를 프레임에 넣어 아트 오브젝트로 판매. 내부 부품이 시각적 구성이 되고, "기술보다 예술에 가까운" 이미지가 된다.

**VDL 적용:**
- **UI 해부(비주얼 아이덴티티 v0.4 §7.2의 "키비주얼 대표 장면")의 물리적 확장.** UI 컴포넌트를 z축 분해한 아이소메트릭 일러스트 → 이것을 실제 아크릴 레이어로 분리 제작 → 물리적 "UI 해부 오브제".
- 분해도(Exploded View)의 역사: 레오나르도 다 빈치(15세기)까지 거슬러 올라감. "중심에서 작은 통제된 폭발이 있었던 것처럼" 구성 요소가 약간 분리된 상태. VDL의 아이소메트릭 z축 분해가 바로 이 전통의 연장선.

#### REF 3-D. 블루프린트 / 테크니컬 드로잉 미학

**핵심:** 2025-2026 디자인 트렌드로 블루프린트 그래픽이 부상. 모든 스트로크가 동일한 웨이트, 2색 스킴, 오브젝트를 레이블된 부품으로 분해. 전자기기, 스킨케어, 의류 패키징에 "분해도 일러스트레이션" 적용.

**VDL 적용:**
- VDL의 모노라인(1px, round cap/join) + 다크 배경 + Naming Line 시스템이 이미 블루프린트 미학의 변주. 이 미학을 물리적 제작물(명함, 스티커, 패키징, 이벤트 공간)로 확장할 때의 소재 선택:
  - **레이저 각인**: 다크 아크릴/금속에 1px 라인을 레이저로 각인 → 모노라인의 물리적 등가
  - **UV 프린팅**: 매트 블랙 위에 글로스 UV로 Grid 패턴 → 빛에 따라 보이는 그리드
  - **핫스탬핑**: violet-tinted 은박 → violet-gray의 물리적 등가

#### REF 3-E. Sagmeister & Walsh — 물리적 타이포그래피 설치

**핵심:** 피부에 직접 타이포그래피를 그리는 작업. 비엔나 400m² 벽화. 물리적 소재(피부, 벽, 오브젝트)가 타이포그래피 메시지와 분리 불가능. "감정과 인간 중심성을 디자인에 되돌리는" 철학.

**VDL 적용:**
- 비주얼 원칙 2: "소재의 온도가 차별화다. 차갑게 느껴지면 틀렸다. 무채색이되 무감정은 아니다."
- 모노라인의 기계적 정밀함을 물리적 소재의 온기로 중화하는 전략. 와이어 벤딩의 손 떨림, 레이저 각인의 미세한 번짐, 종이의 텍스처 — 디지털에서는 0인 이 편차들이 물리적 제작물에서 "온도"가 된다.

#### REF 3-F. MUJI / Kenya Hara — 비움의 물리적 공간

**핵심:** "비움(空)"은 미니멀리즘보다 근본적인 상태. "빈 공간은 환상적인 이미지를 만들 잠재력을 지닌 창작의 그릇." 방문자가 스스로 대화하게 하는 공간. 제품 네이밍도 의도적으로 열려 있다: 용도를 지정하지 않고 소재만 기술("oak table").

**VDL 적용:**
- VDL의 Grid 배경이 MUJI의 "비움"과 같은 역할. Grid는 콘텐츠를 위한 **비어 있는 무대**. 물리적 공간에서도 동일: 이벤트 공간의 벽면에 Grid 패턴만 깔고, 그 위에 참여자의 작업물이 채워지는 구조.
- 빈 Grid = 가능성의 표면. Dictionary 키워드가 채워질 때 비로소 디자인이 되듯, 빈 Grid가 채워질 때 비로소 공간이 완성된다.

#### REF 3-G. Dieter Rams / Braun — 체계적, 파라메트릭 디자인

**핵심:** "Weniger, aber besser" (less, but better). 형태 자체가 문서화. 모든 디테일에 일관성. 가능한 한 적은 디자인. 오브젝트가 "본질로 축소되고 방해 요소가 제거"됨.

**VDL 적용:**
- VDL의 물리적 제작물이 Rams의 원칙과 일치해야 한다: 제거가 크래프트(원칙 1). 물리적 오브젝트에서도 장식 요소 0. 모든 물리적 요소가 의미를 가진다.
- 명함, 스티커, 패키징이 "이 정도면 충분하다"가 아니라 "이 이상은 없앤다"의 결과.

### 3.4 물리적 시각 메타포 체계

| 디지털 모티프 | 물리적 등가 | 소재 | 감각 |
|-------------|-----------|------|------|
| 1px Monoline | 0.5mm 벤딩 와이어 / 레이저 각인 라인 | 스테인리스 와이어, 아크릴 | 정밀, 촉각적 날카로움 |
| Naming Line dot | 핀 헤드 / 엠보싱 도트 | 금속 핀, 형압 | 지점, 기원 |
| Naming Line label | 레이저 각인 모노스페이스 / UV 프린트 | 아크릴, 종이 | 읽을 수 있는 이름 |
| Grid 배경 | 모눈 각인 / UV 패턴 / 천공 | 종이, 아크릴, 금속 | 구조, 가능성의 표면 |
| 아이소메트릭 분해 | 아크릴 레이어 분리 / 와이어프레임 조각 | 투명 아크릴, 와이어 | 해부, z축의 물리적 경험 |
| Circuitry 연결선 | PCB 트레이스 / 벤딩 와이어 경로 | 동선, 와이어 | 연결, 흐름 |
| violet-tinted gray | 매트 블랙 + violet 은박 | 금속, 특수 잉크 | 차가운 듯 따뜻한 |
| 구문 하이라이팅 (유일한 유채색) | 유채색 실크스크린 / 형광 UV | 특수 잉크 | 코드의 유일한 색채 = 활성 |

### 3.5 물리적 적용 시나리오

#### 시나리오 A: 명함

```
앞면:
┌─────────────────────────────────┐
│  (UV 글로스 Grid 패턴, 매트 블랙 위)  │
│                                 │
│         <V>                     │
│         ● ─── vibe-design-labs  │ ← Naming Line (레이저 각인)
│                                 │
│  DDD                            │
│  ● ─── founder / practitioner   │ ← Naming Line
│                                 │
└─────────────────────────────────┘

뒷면:
┌─────────────────────────────────┐
│  (코드 블록, 구문 하이라이팅 유채색)   │
│                                 │
│  <V design="system-over-drawing"│
│     standard="vibe"             │
│     build={true}                │
│  />                             │
│                                 │
│  vibe-design-labs.com           │
└─────────────────────────────────┘
```

- 소재: 매트 블랙 코튼지 600gsm
- 가공: UV 글로스(Grid), 레이저 각인(Naming Line + 텍스트), violet 은박(로고)
- 유채색: 구문 하이라이팅만 실크스크린

#### 시나리오 B: UI 해부 오브제

UI 컴포넌트를 아이소메트릭으로 분해한 장면을 물리적으로 제작:

```
레이어 구성 (아래에서 위로):

[Layer 0] 베이스 — 매트 블랙 아크릴, Grid 레이저 각인
     ↑ 8mm 스페이서
[Layer 1] 배경 — 스모크 아크릴, bg-primary 색상
     ↑ 8mm 스페이서
[Layer 2] 카드 — 반투명 아크릴, bg-secondary
     ↑ 8mm 스페이서
[Layer 3] 텍스트 + 컴포넌트 — 투명 아크릴에 UV 프린트
     ↑
[Wire] Naming Line — 스테인리스 와이어 + 핀 + 모노스페이스 레이블 태그
```

- 레이어 간 8mm 간격 = 디지털 `space-sm`의 물리적 등가
- 30° 기울기로 전시 → 아이소메트릭 투영의 물리적 체험
- Naming Line이 와이어로 각 레이어에 부착되어 `bg-primary`, `card`, `typography.h2` 등을 레이블링

#### 시나리오 C: 코호트 이벤트 공간

```
벽면:
- 매트 블랙 도장 + UV 글로스 Grid 패턴 (8px × 확대 비율)
- Naming Line을 벽에서 물리적으로 돌출 (와이어 + 핀 + 아크릴 레이블)
- 참여자 작업물이 Grid 위에 부착되면서 공간이 완성

테이블:
- 매트 블랙 표면에 Grid 레이저 각인
- 모노스페이스 폰트로 좌석 네이밍: `seat.alpha`, `seat.beta`...

배지:
- 매트 블랙 아크릴, 참여자 이름 레이저 각인
- 제너레이티브 시스템으로 생성된 고유 아이소메트릭 패턴
```

## 4. 세 방향의 교차점 — 키 비주얼 통합 가설

### 4.1 통합 장면

세 방향이 하나의 키 비주얼 장면으로 수렴하는 가설:

```
장면: "디자인 키워드가 코드를 거쳐 디자인이 되는 순간의 아이소메트릭 분해도"

왼쪽 영역 [말]
  ├─ 자연어 프롬프트가 모노스페이스로 타이핑되고 있다
  ├─ 키워드에 Naming Line이 붙어 있다
  └─ 아직 회색 (text-secondary)

중앙 영역 [코드 / 변환]
  ├─ <V> 마크가 변환의 중심에 있다
  ├─ 프롬프트의 키워드들이 코드 구문으로 분해된다
  ├─ Circuitry 연결선이 키워드와 코드를 잇는다
  ├─ 구문 하이라이팅의 유채색이 여기서만 나타난다
  └─ Grid 배경이 가장 선명하다

오른쪽 영역 [디자인]
  ├─ 코드가 아이소메트릭 UI 컴포넌트로 물질화된다
  ├─ z축 분해로 레이어 구조가 노출된다
  ├─ 각 레이어에 Naming Line이 부착되어 토큰명을 표시한다
  └─ 가장 밝다 (text-primary)
```

### 4.2 제너레이티브 변주 규칙

이 장면은 고정이 아니다. 다음 파라미터로 무한 변주된다:

| 파라미터 | 변동 범위 | 결정자 |
|---------|----------|--------|
| 입력 키워드 | Dictionary 전체 어휘 | 콘텐츠 주제 |
| 분해 대상 UI | 컴포넌트 텍소노미 전체 | 코스 모듈 |
| Naming Line 밀도 | 1-8개 | 콘텐츠 깊이 |
| Circuitry 복잡도 | 단순 직선 ~ 다단 꺾임 | 개념 간 관계 수 |
| 컬러 액센트 | 구문 하이라이팅 팔레트 내 | 코드 언어 종류 |
| z축 분해 깊이 | 2-6 레이어 | 컴포넌트 복잡도 |

### 4.3 "Code is Beautiful"이 관통하는 방식

세 방향을 `<V>` 기호가 관통한다:

```
[1] Generative    →  코드(알고리즘)가 시각을 만든다      →  코드 = 도구
[2] Code is Beautiful → 그 코드가 아름답다              →  코드 = 소재
[3] Materialize   →  그 아름다움이 만져진다             →  코드 = 물질
```

`<V>`는 이 세 층위를 모두 축약하는 기호:
- `<` = 코드의 시작점 (도구)
- `V` = 바이브 디자인 (소재)
- `>` = 결과의 출현 (물질)
