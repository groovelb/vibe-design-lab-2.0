**검증:** 브랜딩 전략의 페르소나 배분이 실제 수요와 정확히 일치한다.

## 2. 배우는 이유 (1순위)

| 이유 | 빈도 | 주요 직무 |
|------|------|----------|
| **직무 스킬 업그레이드** | 1위 | 디자이너 압도적 |
| **개인 사업용 IT 서비스 구축** | 2위 | 전 직무 고른 분포 |
| **사이드 프로젝트 역량** | 3위 | 디자이너 + 개발자 |

→ "스킬 업"과 "독립 구축"이 양대 욕구. 브랜딩 전략 §1.5.3 시간축(단기 스킬 → 장기 독립)과 정확히 대응.

## 3. 코스 기대치 (유저의 언어)

| 기대 | 빈도 | 버벌 브랜딩 대응 |
|------|------|-----------------|
| **"정확하고 심미성 높은 결과물"** | 1위 | The Vibe Standard — "정확하게 말하면 정확하게 나온다" |
| **"디자인 시스템 관리 + 코드 기반 디자인 옵스"** | 2위 | System Over Drawing — "기준이 먼저다" |
| **"독립적인 서비스 런칭·운영"** | 3위 | Design As Build — "설계가 곧 구현이다" |

유저가 기대하는 세 가지가 VDL의 세 가치 제안에 1:1로 매핑된다.

## 4. 걱정(페인포인트) 클러스터링

### A. "개발 지식이 없는데 따라갈 수 있을까" — 압도적 1위 (~40%)

> "코딩은 ㅋ자도 모르고" / "개발 지식이 없어서 토큰만 날리게 되는 거 아닐까" / "코드를 이해하지 못한 채 복붙으로 끝날까봐" / "어디서부터 시작해야 될지 감이 안 옵니다"

→ 이 사람들에게 "사고와 구현의 주체를 일치시키는 디자인 언어"는 **문턱이 더 높아 보일 수 있음**

### B. "의도대로 안 나올까" (~15%)

> "디자인한대로 구현이 잘 될지" / "figma로 구현한 디자인 100% 매치" / "막무가내로 프롬프트를 쓰고 구현하다보니 리팩토링에 시간이 훨씬 걸리는 문제"

→ VDL이 정확히 해결하는 문제. **"의도대로 나온다"가 이 사람들의 구매 언어**

### C. "결과물 품질 — AI 티, 완성도" (~12%)

> "AI 티가 많이 나는 획일적인 디자인에서 탈피하고 싶음" / "진짜 디자이너가 한 것처럼 예쁘게" / "production 수준의 디자인" / "부끄럽지 않은 결과물"

→ 디자이너와 개발자 공통. **"정밀"한 결과물에 대한 갈증**

### D. "유지보수 + 운영" (~8%)

> "유지보수를 감당할 수 있을지" / "오류 발생 시 대처" / "디버깅을 혼자 할 수 있을지"

### E. "AI와의 소통 — 프롬프트/디렉션" (~8%)

> "AI가 알아들을 수 있는 '디자인 개발 용어'를 명확히 알고 암기할 수 있어야 한다" / "정확한 디렉션을 못 하는 것" / "개발 용어 지식이 모자라서"

→ **이 클러스터가 VDL의 핵심 타겟.** "디자인 언어"가 바로 이 사람들이 찾는 것

## 5. 유저 언어 vs 브랜딩 언어 대조

| 브랜딩 문서의 언어 | 유저가 실제로 쓰는 언어 |
|-------------------|----------------------|
| 사고와 구현의 주체를 일치시킨다 | "디자인한대로 구현이 잘 될지" |
| 디자인 언어 | "AI가 알아들을 수 있는 디자인 개발 용어" |
| System Over Drawing | "디자인 시스템을 잘 개발해두고 개발하고 싶다" |
| The Vibe Standard | "명확하게 지시할 수 있는 용어를 알면 가능하지만, 없으면 불가능" |
| Design As Build | "기획서 초안 후 2-3시간 안에 워킹 가능한 수준" |
| 넘기지 않고 끝까지 | "넘볼 수 없는 개발자의 영역에 대한 걱정" |
| 주도권 | "혼자서도 만들 수 있는 능력" |

## 6. 버벌 브랜딩 시사점

### 유저 욕구의 3단계 구조

```
표면: "바이브 코딩을 배우고 싶다" (검색 키워드)
     ↓
실제: "정확하고 심미성 높은 결과물을 만들고 싶다" (구매 이유)
     ↓
깊은: "디자인 의도대로 구현되는 경험을 하고 싶다" (감정적 욕구)
```

이 구조가 SWOT GPT의 3층 포지셔닝과 동일하다:
- 표면 카테고리: 바이브 코딩
- 실제 약속: 정확한 결과물
- 궁극 가치: 의도대로 구현

### 현재 Hero와의 긴장

현재 headline "사고와 구현의 주체를 일치시키는 디자인 언어"는 **궁극 가치 층위**에서 말하고 있다. 정확하지만, 유저의 40%가 "개발 지식 없이 따라갈 수 있을까"를 걱정하는 상황에서, 이 밀도가 **"어려운 곳"이라는 인상**을 줄 수 있다.

### 유저 언어에서 추출한 Hero 키워드 후보

| 키워드 | 출현 맥락 | VDL 톤 적합성 |
|--------|----------|-------------|
| **의도대로** | "의도대로 나온다", "디자인한대로" | 화이트리스트 |
| **정확** | "정확하고 심미성 높은", "정확한 디렉션" | 정밀 축 |
| **기준** | "좋은 디자인에 대한 기준조차 없음" | 화이트리스트 |
| **직접** | "직접 만들고 싶다", "혼자서도" | 화이트리스트 |
| **끝까지** | "끝까지 만드는", "100% 매치" | 주도 축 |
